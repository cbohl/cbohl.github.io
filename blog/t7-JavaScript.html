Ruby Classes and JavaScript Functions


Learn how to do the same thing... in two different languages!


March 1, 2015

In programming, it is often useful to build a model that can be used for multiple instances. For example, one might want to create multiples dogs, each which has its own name, age, and tricks it can do. Both Ruby and JavaScript allow the programmer to do this.

Let's start with Ruby. We will define a class, dog.

Input:

class Dog
  attr_reader :name, :age, :tricks

  def initialize(name, age, tricks)
    @name = name
    @age = age
    @tricks = tricks
  end
end

Great. Now let's create an instance of the dog class named Skipper, and print out its contents.

Input:

skipper = Dog.new("Skipper", 10, ["shake", "sit"])

puts skipper.name
puts skipper.age
puts skipper.tricks
Output:
Skipper
10
shake
sit

Success! @name, @age, and @tricks are all instance variables, and therefore are typically inaccessible outside of the class. In this case, however, we use the attr_reader when we create the class so that we can access them.

Now, let's do the same thing in JavaScript.

Input:

function dog(name, age, tricks) {
  this.name = name;
  this.age = age;
  this.tricks = tricks
}

Notice how JavaScript calls this a function, rather than a class. In this situation, however, it will serve the same purpose.

Now, let's create an instance, kit, and print out its contents.

Input:

var kit = new dog("Kit", 4, ["roll-over", "heel"]);

console.log(kit.name)
console.log(kit.age)
console.log(kit.tricks)
Output:
Kit
4
[ 'roll-over', 'heel' ]

Cool. Just like with the Ruby, we have created a new instance and printed its results. Notice how, like with Ruby, we had to make the variables accessible to all. Instead of using attr_reader though, we created separate methods in the fuction by using this.method.

And that is how you can create a model in two languages important to web developers: Ruby and JavaScript.
